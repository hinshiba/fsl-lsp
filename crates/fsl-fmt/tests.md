# fsl-fmt テストリスト

t_wada 流 TDD のテストリスト。1 サイクル (Red → Green → Refactor) ごとに 1 項目を消化し、チェックボックスを ✅ に更新する。新しい必要が判明したら本リストに追記する。

## 公開 API

`pub fn format(src: &str) -> Option<String>`

- 成功時: `Some(整形済みソース)`（末尾は改行 1 つで終わる）
- 失敗時: `None`（パースエラー／レキサーエラーがあれば部分整形しない）

## 正常系

### 最小ケース
- [x] 空ソース `""` → `Some("")`
- [x] 空モジュール `module M {}` を整形して返す
- [x] 空 trait `trait T {}` を整形して返す

### インデント
- [x] モジュール本体は 2 スペースでインデント
- [x] always / initial ブロック内は基底 + 2 スペース
- [x] ネストブロック (`if` / `par` / `seq` / `match`) は深さに応じて 2 スペース × n

### 空白
- [x] 二項演算子 (`+`, `-`, `==`, `&&`, `||`, ほか) の前後にスペース 1 つ
- [x] 単項演算子 (`!`, `-`) と被演算子の間にスペースなし
- [x] カンマの後にスペース 1 つ、前にスペースなし（関数呼び出し / タプルで検証）
- [x] 型注釈の `:` は前にスペースなし、後にスペース 1 つ
- [x] `:=`（reg 代入）の前後にスペース 1 つ

### ブロック・ブレース
- [x] K&R: 開きブレース `{` は前トークンと同じ行
- [x] 閉じブレース `}` は独立行、対応する開きと同じインデント

### 区切り
- [-] セミコロンは省略可能位置でも挿入する（plan.md 準拠）— **MVP 延期**: 一般ブロックでは最終式が値となる可能性があり、無条件挿入は意味を変える恐れがある。再パース可能性は他のテストで担保
- [x] CompilationUnit のトップレベルアイテム間は空行 1 つ

### 改行
- [x] 出力末尾は改行 1 つで終わる（全テストで検証）
- [x] 連続する空行は最大 1 つに圧縮（パーサが改行を破棄するため出力構造的に自明）

### 各構文要素
- [x] `reg name: Bit(8)` 宣言を整形
- [x] `input` / `output` / `mem` 宣言を整形
- [x] `def f(a, b): T = expr` を整形
- [x] `def f(...): T seq { ... }` を整形
- [x] `def f(...): T par { ... }` を整形
- [x] `match` 式: `case pat => body` 各アームを整形
- [x] `if (cond) then-expr` を整形（else なし）
- [x] `if (cond) then-expr else else-expr` を整形
- [x] `val name = expr` 宣言を整形
- [x] タプル型 `(T1, T2)` を整形
- [x] `Array[T]` / `List[T]` / `Bit(N)` 型を整形

### 演算子優先順位
- [x] 必要な括弧は維持、不要な括弧は除去しない（パース木の構造を保つ）

### 行幅
- [x] 100 文字以内に収まる行はそのまま 1 行（既定動作）
- [-] 100 文字超の関数呼び出しは引数で改行（ソフトラップ）— **MVP 延期**: 計測＋分割の実装が必要
- [-] 関数呼び出しのソフトラップは `func(` 直後・`)` 直前で改行しない（FSL パーサ制約） — 上記延期に紐付く
- [-] 引数が 1 行に収まらない場合、第 1 引数は `func(` と同じ行に置き、第 2 引数以降をカンマ直後で改行 — 上記延期に紐付く

### コメント保存
- [x] 単独行の `//` コメントを直前／直後の構文要素の直上行として保持
- [x] 単独行の `/* ... */` コメントを保持
- [x] 式中コメントは整形上の都合で式の前後または直上行に移動してよい（消失させない）
- [x] **式インラインコメントは直上行に移動する（下ではない）**: `val z = a /* tag */ + b` → `/* tag */` を `val z = a + b` の上に
- [x] **ブロック境界 `{ ... /* foo */ }` を越えてコメントが移動しない**: ブロック内末尾のコメントは閉じ `}` の直前に残す
- [x] 行末コメント `} // add` のように直前の `}` が閉じるブロックの「親」スコープにとどまる（次の閉じ `}` までに排出）
- [x] 空ブロック内にコメントがある場合は複数行 `{ コメント }` 形式に展開する
- [x] **物理行末トレイリングコメント `input a: Bit(8) // hoge` を直上行に移動**: AST span の外でも同じ物理行にあるコメントは文に属するとみなす

### サンプル統合
- [x] `fsl-sample/add32.fsl` を整形 → 結果が再パース可能
- [x] `fsl-sample/test_alu32.fsl` を整形 → 結果が再パース可能

## 異常系

- [x] パースエラーあり → `None`
- [x] レキサーエラーあり → `None`
- [x] 不完全モジュール `module M {` → `None`
- [x] 不正トークン (例: `@@@`) → `None`
- [x] 空ソースは異常ではない（`Some("")` を返す）

## MVP 非対応

- 設定可能なスタイル（インデント幅、行幅）
- 整形済みコードと未整形コードの差分計算
- 100 文字ソフトラップ（実装は線分計測＋カンマ分割が必要、別タスク）
- セミコロン挿入（意味論検証が必要、別タスク）
